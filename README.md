# SALES_COUNTER

訪問営業の移動・休憩・ファネルを時刻付きで記録するタップカウンター（スマホ向け Web アプリ）。

- オフィス出発 → 現場到着 → 休憩開始 → 移動開始 → 現場出発 → オフィス到着
- インターホン押下 → インターホン応答 → 対面接触 → 拒否クローズ → アポ取得 → アポ訪問 → プレゼン前拒否 → プレゼン → プレゼン後拒否 → 保留／見込 → セールス
- インターホン押下では新規・既加入・過去解約を選択し、時刻とともに履歴へ保存
- インターホン応答では初回応答・2回目以降を選択して保存
- 対面接触では初回・2回目以降と年代を選択し、時刻とともに履歴へ保存
- アポ取得では対面取得／その他取得を判定し、日付・開始／終了時刻・任意メモを保存
- アポ訪問では予定アポ・当日取得アポで絞ったリストから対象を選び、対象がなければアポ取得フローを挿入
- 前提イベントが直前にない場合は選択ポップアップを挟み、10秒刻みでインターホン押下・応答・対面接触・アポ取得・訪問・プレゼンを自動補完
- 3種類の拒否では拒否理由を選択し、「その他」は自由入力して履歴へ保存
- 保留／見込では見込度1〜5と任意コメントを保存
- カウンター／アポ／保留・見込のタブ切り替えと、各リストの GPS ボタン
- ハンバーガーメニューの設定から GPS をON/OFF（初期値OFF）し、ON時のみ全活動で現在地を内部保存
- 履歴ログを整形してクリップボードへコピー
- 分析ボタンから営業ファネル件数と各コンバージョン率を表示
- 活動終了時に当日の活動を日報として保存し、日報タブから集計結果・タイムラインを確認
- タップするだけで日次のファネル数値を記録
- 履歴表示 / 全リセット / 直前の 1 タップを取り消し
- localStorage 永続化、オフラインでも動く
- 20:9 のスマートフォン基準フレーム内で、3パートとボタン寸法を比例配置

## Live

https://gridatlas.github.io/SALES_COUNTER/

## 技術スタック

Next.js 14 (App Router, static export) / TypeScript / Tailwind CSS / Zustand + persist / lucide-react

## 開発

```bash
npm install
npm run dev          # http://localhost:3000
npm run build        # 静的エクスポート → out/
```

## デプロイ

`main` に push すると GitHub Actions が `out/` を GitHub Pages に自動デプロイ（`.github/workflows/deploy.yml`）。

## 削除運用

localStorage キー: `sales-counter-store`、`sales-counter-settings`。ブラウザの開発者ツール → Application → Local Storage から手動削除も可能。
